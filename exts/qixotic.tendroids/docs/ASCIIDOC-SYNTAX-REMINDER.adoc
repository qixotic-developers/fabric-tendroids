= ASCIIDOC SYNTAX REMINDER
:date: 2025-11-27

== ⚠️ CRITICAL: AsciiDoc vs Markdown Syntax

When creating `.adoc` files, ALWAYS use AsciiDoc syntax, NOT Markdown!

== Common Mistakes to Avoid

[cols="2,2,1"]
|===
|Element |❌ Markdown (WRONG) |✅ AsciiDoc (CORRECT)

|*Bold text*
|`pass:[**text**]`
|`pass:[*text*]`

|_Italic text_
|`pass:[*text*]`
|`pass:[_text_]`

|Code block
|````python` +
code +
[source,`]
-----
|`[source,python]` +
`----` +
code +
`----`

|Inline code
|``pass:[`code`]``
|``pass:[`code`]``  _(same)_

|Headings
|`# ## ###`
|`= == ===`

|Lists
|`- item` or `* item`
|`* item` or `- item`  _(context matters)_

|Links
|`[text](url)`
|`link:url[text]` or `https://url[text]`

|Tables
|`\| --- \|`
|`\|===`
|===

== Quick Reference

*Bold*: Single asterisk `*bold*`

_Italic_: Single underscore `_italic_`

*Code blocks*:
[source,python]
----
[source,python]
----
your code here
----
----

*Headers*:
[source,asciidoc]
----
= Document Title (level 0)
== Section (level 1)
=== Subsection (level 2)
==== Sub-subsection (level 3)
----

*Lists*:
[source,asciidoc]
----
* Unordered item
* Another item
  ** Nested item

1. Ordered item
2. Another item
----

== When to Use AsciiDoc

- `.adoc` files → ALWAYS use AsciiDoc syntax
- `.md` files → Use Markdown syntax
- Documentation in code → Usually AsciiDoc for Nvidia/Omniverse projects

== Testing

Before committing `.adoc` files, verify:
1. No `pass:[**double asterisks**]` for bold
2. No ````backtick code blocks````
3. No `# markdown headings`
4. Proper `[source,language]` + `----` blocks

== Tools

To validate AsciiDoc syntax:
- Visual Studio Code: Install "AsciiDoc" extension
- Command line: `asciidoctor --safe-mode=safe file.adoc`
- Preview: Most extensions show "Markdown style listing" warnings

== Remember

**If you see "Markdown style listing" warnings in the editor, YOU USED MARKDOWN SYNTAX IN AN ASCIIDOC FILE!**

Fix it before committing!
