== Issues Fixed

=== Issue 1: Warp Cylinder Extension Still Loading

**Problem**: Despite setting `enabled = false`, the extension was still loading and causing errors

**Root Cause**: The extension was being discovered from the extensions folder before the settings could disable it

**Solution**: Renamed the extension folder to disable discovery
- From: `omni.example.warp.cylinder`
- To: `omni.example.warp.cylinder.disabled`

**Location**: `C:\Dev\Omniverse\omni-tendroids\source\extensions\`

**Result**: Extension will no longer be discovered or loaded

=== Issue 2: Missing test_scenarios Module

**Error**: `ModuleNotFoundError: No module named 'qixotic.tendroids.warp_test.test_scenarios'`

**Root Cause**: `cpp_batch_test_controller.py` was importing `BATCH_15_CPP_SCENARIO` from the archived `test_scenarios.py`

**Solution**: Inlined the configuration constants instead of importing

**File Modified**: `qixotic/tendroids/warp_test/cpp_batch_test_controller.py`

**Changes**:
[source,python]
----
# Before - importing from archived module
from .test_scenarios import BATCH_15_CPP_SCENARIO

# After - inlined constants
BATCH_15_MAX_FRAMES = 2000
BATCH_15_TUBE_COUNT = 15
BATCH_15_SEGMENTS = 16
BATCH_15_RADIAL_SEGMENTS = 12
----

**Updated References**:
- `BATCH_15_CPP_SCENARIO.max_frames` → `BATCH_15_MAX_FRAMES`
- `scenario.cylinder_count` → `BATCH_15_TUBE_COUNT`
- `scenario.segments` → `BATCH_15_SEGMENTS`
