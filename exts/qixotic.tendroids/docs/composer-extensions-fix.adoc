= Additional Composer Fixes - Extensions
:toc:
:toclevels: 2

== Issues Addressed

=== Broken Warp Cylinder Extension

**Error**: `omni.example.warp.cylinder` failing to load with import error

**Solution**: Disabled the broken example extension

**File Modified**: `source/apps/qixotic.usd_composer.kit`

**Change**:
[source,toml]
----
[settings.exts]
"omni.example.warp.cylinder".enabled = false  # Broken example extension
----

This prevents the error from appearing and doesn't affect Tendroids functionality.

=== Tendroids Extension Not Visible

**Problem**: `qixotic.tendroids` extension not appearing in Extensions window

**Root Cause**: Extension path not in Composer's search folders

**Solution**: Added Tendroids extension path to folders list

**File Modified**: `source/apps/qixotic.usd_composer.kit`

**Change**:
[source,toml]
----
[settings.app.exts.folders]
'++' = [
    "${app}/../exts",
    "${app}/../extensions",
    "${app}/../extscache",
    "C:/Dev/Omniverse/fabric-tendroids/exts"  # Added for Tendroids
]
----

== Extension Discovery

With this configuration, Composer will now scan:

1. **Local extensions** (in omni-tendroids repo):
   - `source/exts/`
   - `source/extensions/`
   - `source/extscache/`

2. **Tendroids extension** (in fabric-tendroids repo):
   - `C:/Dev/Omniverse/fabric-tendroids/exts/`
   - This contains `qixotic.tendroids`

== Extensions Window Behavior

After restart, Extensions window will:

✅ Default to "qixotic" search filter
✅ Default to "THIRD PARTY" category
✅ Show `qixotic.tendroids` extension
✅ Allow enable/disable of Tendroids extension
✅ NOT show broken `omni.example.warp.cylinder`

== Enabling/Disabling Tendroids

**In Extensions Window:**
1. Search shows "qixotic" by default
2. Find `qixotic.tendroids` in list
3. Toggle switch to enable/disable
4. Extension loads with Tendroid Controls window

**In Kit File:**
If you want to always enable it on startup, add to dependencies:
[source,toml]
----
[dependencies]
"qixotic.tendroids" = {}
----

== Alternative: Symbolic Link

Instead of hardcoding the path, you could create a symbolic link:

**Windows Command (run as Administrator):**
[source,batch]
----
mklink /D "C:\Dev\Omniverse\omni-tendroids\source\extensions\qixotic.tendroids" "C:\Dev\Omniverse\fabric-tendroids\exts\qixotic.tendroids"
----

This creates a link in the local extensions folder pointing to your Tendroids extension.

**Benefits:**
- No hardcoded path in kit file
- Works consistently across environments
- Extension appears in standard location

**Drawback:**
- Requires admin privileges to create
